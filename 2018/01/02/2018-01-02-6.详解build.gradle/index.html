6.详解build.gradle

发表于 2018-01-02 |
字数统计: | 阅读时长 ≈

build.gradle

不同于Eclipse，AndroidStudio采用Gradle构建项目，
它使用一种基于Groovy声明项目设置，摒弃XML繁琐设置

外层build.gradle

buildscript{
    repositories{
        jcenter()
    }
    dependencies{
        classpath 'com.android.tools.build:gradle:2.2.0'
    }
}
allprojects{
    repositories{
        jcenter()
    }
}

两处repositories都声明了jcenter(),jcenter()是一处代码托管仓库
很多Android开源项目都将代码托管到jcenter，声明这行配置，就可以在项目引用jcenter的开源项目

dependencies使用classpath声明了一个Gradle插件，
为什么声明这个插件？
因为Gradle并不是专门为构建Android而开发的，Java、C++很多项目都可以使用Gradle构建
如果我们想使用它构建Android项目，需要声明com.android.build:gradle:2.2.0这个插件

app目录下的build.gradle

apply plugin: 'com.android.application'

android{
    compileSdkVersion 24
    buildToolsVersion "24.0.1"
    defaultConfig{
        applicationId "com.example.helloworld"
        minSdkVersion 15
        targetSdkVersion 24
        versionCode 1
        versionName "1.0"
    }
    buildTypes{
        release{
            minifyEnabled false
            proguardFiles getDefaultProguardFile('proguard-android.txt',
                'proguard-rules.pro'
        }
    }
}
dependencies{
    compile fileTree(dir: 'libs', include: ['*.jar'])
    compile 'com.android.support:appcompat-v7:24.2.1'
    testCompile 'junit:junit:4.12'
}

apply plugin有两种选择：com.android.application/library，
一个是应用程序模块，一个是库模块。应用模块可以直接运行，库模块只能作为代码库依赖于别的应用程序。

android闭包

compileSdkVersion，指定项目编译版本
buildToolsVersion，指定项目构建工具的版本

#defaultConfig闭包
applicationId，指定项目包名
minSdkVersion，指定项目最低兼容的Android版本
targetSdkVersion，目标版本
versionCode，指定项目的版本号
versionName，指定项目的版本名

#buildTypes闭包，通常有两个子闭包：debug、release
#debug：测试版，可以忽略不写、release：正式版
minifyEnabled,是否对项目代码混淆
proguardFiles，指定混淆规则

dependencies闭包
指定当前项目所有的依赖关系：本地依赖、库依赖、远程依赖

compile fileTree，本地依赖声明，它表示将libs目录所有.jar添加到项目
com.android.support:appcompat-v7:24.2.1，jcenter远程依赖库，Gradle构建项目会检查本地是否缓存这个库，没有会自动下载
compile project(':adf'),库依赖
testCompile，声明测试用例库
