1.PSTN与VoIP基础

发表于 2017-12-23 |
字数统计: | 阅读时长 ≈

VoIP: Voice Over IP

IP网上的语音通信.

PSTN: Public Switched Telephone Network

公共交换电话网,我们日常打电话所使用的网络

1.1 PSTN起源与发展

1)最早的电话网

    振铃电话: 用户之间用物理线路连接,并且同一时间只能有一个用户讲话.发话方通过话音震动产生电信号,

    电信号传到远端通过震动对方的扬声器发声

    个体之间都需要单独的物理线路,3个人就需要3*(3-1)/2对电话线

2)人工电话交换

    交换机: 位于整个电话网的中心,用户打电话先连接到管理交换机的接线员,由接线员接通到对方线路

3)自动电话交换
    由用户话机的拨号脉冲直接控制交换机的动作

    后来又出现了旋转式和升降式,由记发器接受用户的拨号脉冲,然后将其通过译码器译成电码控制接线器的动作

4)半电子交换机

    电子交换机,控制部分引入电子技术,话录部分仍使用机械触点

5)空分交换机

    在交换机中引入了"存储程序控制"的概念

6)数字交换机

    将"模拟"信号数字化,提高了通话质量,传输距离

7)PSTN

    中继线(Trunk): 连接交换机,PSTN把世界的人们联系在一起,一个通话需要穿越很多交换机

    蜂窝式移动电话,无线电话发展的里程碑

    移动网: 移动电话交换的通信网络

    固网: 原来的程控交换网

    简单来说,移动网就是在普通固网的基础上增加了很多基站(天线),位置寄存器,拜访位置寄存器,记录用户的位置

8)VoIP

    NGN: 基于电路交换的语音网络与基于分组交换的英特网进行融合(即语音通信和数据通信相结合)

    人们提出了各种NGN的解决方案,但最终统一到IMS(IP Multimedia Subsystem)技术,IMS属于核心交换层的技术,全部基于IP网络

    但目前的语音大部分还是基于电路接入的方式,因此,这就要求在一定时间内IMS在接入层要继续兼容电路接入.

    随着3G,4G技术的发展,未来的通信中要完全取消低效的电路传输及电路交换,而全部集中到IP通信,也就催生了新的无线通信标准LTE.

    LTE: 为现代手机及其他移动设备提供高速的数据通信,逐步实现全IP交换.

1.2 电话实现技术

1)电话号码

    固定电话号码: 大省市,8位数字组成,小省市,7位数字

    以ABCDEFGH为例,其中ABCDE称为一个千群,即该群可以包含1000个号码,ABCD称为一个万群,

    小规模 的交换机可能只包含几个千群,大规模的可以包含几个万群.

    不同省市之间打电话,则称为长途电话,一般需要长途局进行路由,打长途需要在本地号码前加上长途区号

2)移动电话和专用号段

    我国移动电话都是以1开头,移动:135,136,137,138,139等,联通:130,186等.它们后面都是8位的号码,前3位称为移动电信运营商专用的号段

3)短号码

    110,119,120等

4)800和400

    企业集团使用,这类号码都是虚拟的电话号码,在实际呼叫过程中通过查询数据库转换成真正的目的号码

    800号码有一个致命的缺陷,用手机打不通,这是由于不同运营商的网间结算造成的.

5)模拟信号与数字信号

    早期的电话网也是基于模拟交换的,对人类交流来讲模拟信号非常理想,但它容易引入噪声,如果通话双方的距离很远,信号会衰减

    数字信号是不连续的,按一定的时间间隔对模拟信号进行抽样得出离散值,然后通过量化和编码将这些离散值变成数字信号

6)PCM

    PCM: 脉冲编码调制,它是一种通用的将模拟信号转换成0和1表示的数字信号的方法

1.3 我国电话网结构

本地网与长途网,并通过国际交换中心进入国际电话网

1.4 信令

话机与端局交换机之间,以及交换机与交换机之间需要进行通信,这些通信所包含的信息有:

用户,中继线状态,主叫号码,被叫号码,中继路由的选择等,我们把这些消息称为指令.


信令功能分类,

    线路信令:监视主被叫,接挂机状态,设备忙闲

    路由信令:主叫所拨的被叫号码,用来选择路由

    管理信令:电话网的管理和维护

信令的工作区域分类,

    用户线信令: 用户与交换机之间的信令,用户状态信号,用户拨号所产生的数字信号,以及交换机向用户发送的信号

    居间信令: 交换机与交换机之间的信令,用来控制呼叫接续盒拆线


H.323与SIP信令

H.323与SIP属于VoIP领域的通信信令,它们适用于用户线信令和居间信令,由于IP终端比普通话更智能,因此这些信令在用户线信令以及局间信令使用方式上没有太大区别

1)H.323属于多媒体通信,在现有通信网络上进行视频会议

2)SIP(Session Initiation Protocol),会话发起协议,用于发起会话
